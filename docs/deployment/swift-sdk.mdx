Native Swift SDK for bitHuman's audio-driven avatar animation. Everything runs on-device — no cloud calls during inference. Mirrors the [Python `AsyncBithuman`](/deployment/self-hosted-gpu) API so the same mental model ports between stacks.

**Repo:** [github.com/bithuman-product/bithuman-sdk-swift](https://github.com/bithuman-product/bithuman-sdk-swift)

## Requirements

- macOS 14+
- **Apple Silicon, M3 or later** (the 1.3 B-parameter DiT needs M3's GPU bandwidth for sustained 25 FPS)
- 16 GB RAM
- ~5 GB free disk (model weights cache)
- Xcode 16.3+ (Swift 6.1 tools)

## 1. Install

Add to your `Package.swift`:

```swift
dependencies: [
    .package(url: "https://github.com/bithuman-product/bithuman-sdk-swift.git", from: "0.1.0"),
],
targets: [
    .executableTarget(name: "MyApp", dependencies: [
        .product(name: "BithumanAvatar", package: "bithuman-sdk-swift"),
    ]),
]
```

## 2. Quick Start

```swift
import BithumanAvatar
import AVFoundation

// Pipeline backing store — holds MLX pipeline + VAE encoder/decoder.
let box = PipelineBox()

// Paths to the four weight artifacts.
let paths = ModelPaths(
    ditWeights: "/path/to/dmd2_run9.safetensors",
    wav2vecWeights: "/path/to/wav2vec2.safetensors",
    refLatent: "/path/to/default_ref_latent.npy",
    aneDecoder: "/path/to/turbo_vaed_ane_384.mlpackage",
    vaeEncoder: "/path/to/vae_encoder.safetensors",
    nSteps: 2
)

let (_, err) = PipelineOps.load(box: box, paths: paths)
if let err = err { fatalError("load failed: \(err)") }

// Create the runtime.
let bithuman = Bithuman(pipelineBox: box)
try await bithuman.start()

// Feed audio — the SDK takes both the 24 kHz display slice and
// the 16 kHz wav2vec2 input. They cover the same time span.
await bithuman.pushAudio(audio24k: samples24, audio16k: samples16)

// Pull frames (audio-clocked — emit as audio is consumed).
while let chunk = bithuman.tryDequeueChunk() {
    for frame in chunk.frames {
        myDisplayLayer.enqueue(frame)         // CGImage at 25 FPS
    }
    if let audio = chunk.audio24k {
        myAudioEngine.schedule(audio)         // 24 kHz Float32 mono
    }
}

// Signal utterance end, or cut off mid-stream.
await bithuman.flush()
await bithuman.interrupt()
await bithuman.shutdown()
```

## 3. API Surface

| Type | Role |
|------|------|
| `Bithuman` (actor) | Primary entry point. Owns the streaming pipeline and audio-dispatch state. |
| `PipelineOps` | Static `load` / `processChunk` / `swapIdentity` operations. Throws typed `PipelineError`. |
| `PipelineBox` | Backing ML state container (MLX pipeline + VAE encoder/decoder + chunk queue). |
| `ModelPaths` | File paths for the four weight artifacts. |
| `TimedChunk` | Output unit — paired video frames + 24 kHz audio slice. |
| `PipelineError` | Typed error cases. |

Full API reference lives in the repo as DocC: `swift package generate-documentation --target BithumanAvatar`.

## Python ↔ Swift Parity

The Swift SDK mirrors the Python `AsyncBithuman` vocabulary so Python developers reading Swift feel at home.

<CodeGroup>
```python Python
runtime = await AsyncBithuman.create(model_path, api_secret)
await runtime.start()
await runtime.push_audio(audio_bytes, sample_rate=16000)
async for frame in runtime.run():
    display(frame.bgr_image)
    play(frame.audio_chunk)
await runtime.flush()
```

```swift Swift
let bithuman = Bithuman(pipelineBox: box)
try await bithuman.start()
await bithuman.pushAudio(audio24k: s24, audio16k: s16)
while let chunk = bithuman.tryDequeueChunk() {
    for frame in chunk.frames { display(frame) }
    if let a = chunk.audio24k { play(a) }
}
await bithuman.flush()
```
</CodeGroup>

**Swift additions** not in Python:
- `interrupt()` — mid-utterance abort (barge-in UX).
- `shutdown()` — explicit GPU-memory release.
- `snapshot` — nonisolated dispatch-state view for sync display ticks.

## Design Principles

- **Audio-clocked.** Frames emit as audio is consumed. The caller owns scheduling — no wall-clock timer inside the SDK.
- **MLX is invisible.** No MLX types leak into the public surface.
- **One MLX eval at a time.** `Bithuman` serializes all DiT / wav2vec2 / VAE calls through a single actor. Concurrent `mlx_eval` calls into MLX's C++ core are the documented cause of crashes; the API prevents you from hitting them.
- **Bounded memory.** Session working set caps ~4 GB during a burst; `shutdown()` returns to baseline.
- **Halo is consumer zero.** [bitHuman Halo](https://github.com/bithuman-product/halo) — a macOS desktop companion — was the first app built on this SDK and pins it as its sole avatar dependency. Any API change lands in the SDK repo first.

## Performance Targets (M3)

| Metric | Target |
|--------|--------|
| First-frame latency (full receptive field) | ≤ 1.3 s |
| First-frame latency (partial window) | ≤ 450 ms |
| Steady-state per-frame (p50 / p99) | ≤ 20 ms / ≤ 40 ms |
| Peak MLX working set during burst | ≤ 4 GB |
| Peak RSS | ≤ 5 GB |

See [`docs/PERFORMANCE.md`](https://github.com/bithuman-product/bithuman-sdk-swift/blob/main/docs/PERFORMANCE.md) in the SDK repo for the measurement protocol.

## Model Weights

The SDK downloads ~3.7 GB of pre-trained weights (DiT + wav2vec2 + VAE encoder + ANE decoder) on first use, cached at `~/Library/Caches/com.bithuman.sdk/weights/`. Hosted on Cloudflare R2. Weights are distributed under a separate license from the code.
